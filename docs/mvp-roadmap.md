# StefaLoveApp MVP Roadmap

## Product Core

The beta should prove one thing first: couples return because the app helps them ask for attention, care, and warmth at the right moment.

Core promise:

> A private space for a couple: SOS kisses, mood check-ins, shared wishes, small games, and a memory of how they care for each other.

## Beta 0.1

Must ship:

- Two real accounts connected into one couple.
- Boyfriend creates the couple and invite code.
- Girlfriend joins by invite code.
- Couple profile: names, relationship start date, personal nicknames.
- SOS requests:
  - kiss
  - hug
  - attention
  - sad/support
  - custom text
- SOS responses:
  - coming
  - in 10 minutes
  - virtual kiss
  - cannot now
- Days-together counter.
- Mood check-in with daily history.
- Shared wishes with statuses.
- Basic pair statistics:
  - SOS sent
  - SOS answered
  - mood check-ins
  - wishes completed
  - games played
  - points earned
- 2-3 games from the current prototype.
- Letters/time capsules from the current prototype.

Do not ship yet:

- Subscriptions.
- Instagram integration.
- Full Telegram integration beyond a simple optional notification bridge.
- 8+ games.
- Complex AI flows.
- Full redesign perfection.

## Beta 0.2

- Daily care quests.
- Better pair achievements.
- Mood insights: "today she/he needs..."
- Gift shop polish.
- Telegram duplicate notifications.
- Theme packs.
- Better onboarding copy.
- Analytics events.

## Beta 0.3

- RevenueCat subscriptions.
- Free/Premium limits.
- Extended statistics.
- iOS widget planning.
- Privacy/export/delete-account flows.
- App Store/TestFlight preparation.

## Swift Trigger

Move to native SwiftUI only after the web beta shows retention:

- 10-20 real couples connected.
- At least 50% send one SOS.
- At least 30% return on the next day.
- At least 5 couples use mood or wishes for 3+ days.

Until then, keep testing on GitHub Pages with Supabase.
